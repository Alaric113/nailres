📌 專案總覽

這是一個為美甲/美睫工作室開發的預約管理系統，名為 TreeRing（品牌名）。 專案使用 React + TypeScript + Vite 前端框架，搭配 Firebase（Auth + Firestore + Storage + Messaging） 作為後端服務，並部署在 Netlify 上（含 Serverless Functions）。

支援 LINE 與 Google 登入、多設計師管理、服務分類/選項、季卡（Season Pass）、 優惠券、集點卡、商品卡（Gift Card）、作品集展示、顧客評價、以及完整的後台管理功能。

⏳ 目前狀態：已上線運作中，但由於缺乏前期規劃，程式碼存在大量結構性問題。

🔐 認證流程

LINE 登入流程

用戶點「LINE 登入」 │ ├─ 判斷是否在 LINE 內建瀏覽器 (LIFF)？ │ ├─ Yes → 導向 /liff?redirect=xxx │ │ → LiffEntry.tsx 初始化 LIFF │ │ → liff.login() → LIFF 取得 LINE Token │ │ → POST /api/line-liff-auth { idToken } │ │ → Firebase signInWithCustomToken │ │ │ └─ No → 判斷是否 PWA？ │ ├─ Yes (PWA) │ │ → 開啟 Popup 視窗進行 OAuth │ │ → 在 Firestore temp_auth_tokens/{state} 監聽 Token │ │ → Popup 完成後寫入 token → onSnapshot 收到 → signInWithCustomToken │ │ │ └─ No (一般瀏覽器) │ → 直接 redirect 到 LINE OAuth URL │ → 回到登入頁面 (code + state) │ → POST /api/line-oauth-auth { code, redirectUri, state } │ → Firebase signInWithCustomToken │ └─ 成功後 useAuth 監聽到 Firebase User → Firestore onSnapshot users/{uid} → 若不存在則 createUserProfile（含新用戶優惠券） → setAuthState(user, profile)

Google 登入（管理員/設計師限定）

  • 僅允許 admin、manager、designer 角色使用
  • 一般用戶使用 Google 登入會被拒絕並刪除 Auth 記錄（避免孤兒帳號）

自動註冊流程

onAuthStateChanged → user 存在但 users/{uid} 不存在 → createUserProfile(): 1. writeBatch 開始 2. 查詢 isNewUserCoupon === true 的優惠券 3. 若有 → 在 user_coupons 建立一筆新用戶優惠券 4. 在 users/{uid} 建立使用者文件 5. batch.commit() → onSnapshot 再次觸發 → data 已存在 → setAuthState(user, profile)

認證流程問題

  • BUG 註冊邏輯有兩處實作：Register.tsx 和 useAuth.ts，可能重複發放新用戶優惠券
  • WARN createUserProfile 使用 batch.set()（非 merge），若因 race condition 雙方同時寫入可能遺失資料
  • WARN ProtectedRoute 使用 console.log 輸出除錯資訊到 production

🧮 完整結帳價格計算流程

定義於 src/pages/BookingPage.tsx 第 138-180 行 — useMemo 計算

計算順序 (由前往後疊加)：

【STEP 1】Base Price（原始總價）
  totalBasePrice = Σ cart[item].totalPrice
                 = Σ (item.service.price + Σ(optionItem.price × qty))

【STEP 2】Platinum 會員折扣（若 role === 'platinum'）
  對每個 cart item:
    serviceDef = services.find(s.id === item.service.id)

    CASE A: serviceDef.platinumDiscount.type === 'percentage'
      // ★ 折扣僅套用於服務原價 servicePrice，加購選項維持原價
      → servicePrice = item.service.price
      → optionsPrice = item.totalPrice - servicePrice
      → itemFinalPrice = Math.floor(servicePrice × (value / 100)) + optionsPrice
      // value = 折扣後百分比 (e.g., 70 = 70% = 打7折)
      
    CASE B: serviceDef.platinumDiscount.type === 'fixed'
      // ★ 折扣僅從服務原價扣除，加購不受影響
      → servicePrice = item.service.price
      → optionsPrice = item.totalPrice - servicePrice
      → itemFinalPrice = Math.max(0, servicePrice - value) + optionsPrice
      // value = 固定折扣金額

    CASE C: serviceDef.platinumPrice (deprecated, 向後兼容)
      → optionsPrice = item.totalPrice - item.service.price
      → itemFinalPrice = serviceDef.platinumPrice + optionsPrice
      // 僅取代基礎價格，options 價格不變

  totalDiscountedPrice = Σ itemFinalPrice
  platinumDiscountAmount = totalBasePrice - totalDiscountedPrice

【STEP 3】優惠券折扣 (Coupon Discount)
  // 僅當 selectedCoupon !== null 且 totalDiscountedPrice >= minSpend

  if (selectedCoupon.type === 'fixed')
    couponDiscount = selectedCoupon.value  // 固定折 N 元

  if (selectedCoupon.type === 'percentage')
    couponDiscount = Math.floor(totalDiscountedPrice × (value / 100))
    // value = 折扣百分比 (e.g., 20 = 20% off)

  finalPrice = Math.max(0, totalDiscountedPrice - couponDiscount)
  totalDiscountAmount = platinumDiscountAmount + couponDiscount

【STEP 4】Result
  finalOutput = {
    totalDuration,     // 總服務時長 (分鐘)
    originalPrice,     // 未折扣原始價格
    finalPrice,        // 最終實付金額
    discountAmount     // 總折扣金額
  }
